A — Option Matrix (Auto-generate)

Seller defines option names (e.g. Size, Color) and values (e.g. S, M, L). The system auto-generates all combinations as variants. Seller then sets price/stock per variant row.

  • Matches Medusa's native data model exactly
  • Fast for simple option sets
  • Can explode with many combinations

B — Manual Variant Rows

Seller types each variant name directly (e.g. "Small / Red") and fills in price, stock, and SKU per row. No auto-generation — full manual control.

  • Simpler to build
  • Flexible — any variant naming scheme
  • Tedious for products with many combinations

C — Skip Variants for Now

Single price + single stock quantity. Simplest possible product model. Variants designed and built in a follow-up sprint once CRUD is stable.

  • Fastest path to a working storefront
  • Requires a migration later to add variants
  • Limits sellers to single-SKU products
